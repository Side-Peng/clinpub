## 简介
本指南面向参与 clinpub 项目的开发者，提供从零搭建开发环境的完整步骤。内容涵盖系统要求、R 语言统计分析包安装、Python 虚拟环境与依赖、Claude Code 技能安装、开发工具配置以及常见问题排查。clinpub 是面向 SCI Q1/Q2 期刊的端到端临床数据分析与发表加速器，采用分阶段工作流（init → data-prep → analysis → writing → review），结合多智能体协作与钩子机制保障流程质量。

## 系统要求
- Claude Code：≥ 2.1.88（技能宿主环境）
- Node.js：≥ 22.0.0（安装器与钩子执行）
- R：≥ 4.2（统计分析）
- Python：≥ 3.9（数据画像、搜索脚本）

上述要求在多个官方文档中得到确认，安装器还会对 Node.js、R、Python 进行软性环境检查，并提示下载链接。

## Python 开发环境
### Python 包与版本
- Python 版本：≥ 3.9
- 必需包：pandas, numpy, requests, openpyxl
- 其他：tavily-python（用于 Tavily 搜索）、pymupdf（用于 PDF 处理）

### 虚拟环境与依赖安装
推荐使用 Python 虚拟环境隔离依赖，安装命令如下：
- 创建虚拟环境：python -m venv .venv
- 激活虚拟环境（Linux/Mac）：source .venv/bin/activate
- 激活虚拟环境（Windows）：.venv\Scripts\activate
- 安装依赖：pip install -r requirements.txt

requirements.txt 中的版本下限可在以下文件中找到：
- requirements.txt：pandas、numpy、requests、openpyxl、tavily-python 的最小版本要求
- docs/CONFIGURATION.md：requirements.txt 的具体版本下限

## Claude Code 与 Node.js 环境

- 使用 npx 一键安装：npx clinpub@latest

## 常见问题排查
### 环境检查与版本问题
- Node.js 版本过低：安装器会提示下载新版 Node.js（≥ 22.0.0）
- R 未安装或不在 PATH：提示下载 CRAN R（≥ 4.2）
- Python 未安装或不在 PATH：提示下载 Python（≥ 3.9）

### 技能与钩子问题
- 技能未出现：重启 Claude Code 以重新加载
- 钩子未生效：确认 settings.json 中已注册三个钩子
- 阶段访问被阻：检查 Milestone 是否完成，确保 Phase 顺序正确

### R 包与 Python 包问题
- R 包安装失败：运行 INSTALL.md 中的 install.packages 命令
- Python 导入错误：运行 pip install -r requirements.txt
- API 密钥问题：设置 NCBI_API_KEY 或 TAVILY_API_KEY 环境变量

## 性能考虑
- R 性能优化：使用 data.table 替代 data.frame；必要时使用并行处理（parallel 包）
- Python 性能优化：大文件分块读取（chunksize）；多进程处理（multiprocessing.Pool）
- 图表输出：遵循出版级标准（≥300 DPI，PNG/PDF/TIFF），字体与配色统一

## 故障排除指南
### 环境与依赖
- Node.js 版本过低：升级至 ≥ 22.0.0
- R 包缺失：使用 INSTALL.md 中的 install.packages 命令批量安装
- Python 依赖缺失：使用 requirements.txt 安装

### Claude Code 与钩子
- 技能未加载：重启 Claude Code
- 钩子冲突：检查 settings.json 中的 PreToolUse 条目，确保三个 clinpub 钩子已正确注册

### 配置与路径
- project_config.yml 字段为空：确认已执行初始化流程，或手动补充关键字段
- 自定义路径不生效：检查 hooks/clinpub-phase-boundary.sh 是否依赖硬编码路径，必要时调整为读取配置
