## Objective

设计统一的「方法说明」模板，将所有引用 README 的管线文档更新为「方法说明」，实现 WAVE 文档中文本地化。确保与现有 MANIFEST.yaml 和 agent-contracts.md 的兼容性。

## Tasks

### Task 1: 设计「方法说明」模板

<read_first>
- `pipeline/references/agent-contracts.md` — 当前 README 输出规范
- `pipeline/references/analysis_methods.md` — 每方法需要 README 的规范
- `.clinpub/phases/06-图表文档优化/06-RESEARCH.md` — 方法说明模板设计（§5）
- `.clinpub/phases/06-图表文档优化/06-CONTEXT.md` — D-08 决策
</read_first>

<action>
在 `pipeline/templates/` 目录下创建 `method-readme.md` 模板文件，内容结构：

```
# [方法名称] — 方法说明

## 目的
[一句话说明本分析的目标]

## 方法
[使用的统计方法、公式、假设条件]

## 输入数据
- 数据文件：`cleaned.csv` / `full_longitudinal.csv`
- 使用变量：[变量列表]
- 样本量：[N]
- 时间点：[baseline / post_treatment / follow_up]

## 输出结果
### 图表
| 文件 | 说明 |
|------|------|
| figure_1.png | [图例说明] |

### 表格
| 文件 | 说明 |
|------|------|
| table_1.docx | [表注说明] |

## 参数设置
- [关键参数及其取值]
- [如：显著性水平 α=0.05, 校正方法 FDR]

## 注意事项
- [方法适用条件]
- [不适用条件]
- [已知局限性]

## 软件版本
- R {version}
- {包名} {版本}
```

模板中"输出结果"section 等价于原 "Results" subsection，保持兼容性。
</action>

<acceptance_criteria>
- `pipeline/templates/method-readme.md` 文件存在
- 模板包含：目的、方法、输入数据、输出结果、参数设置、注意事项、软件版本
- "输出结果"section 包含图表和表格两个子表格
- 模板使用中文，统计术语和 R 包名保持英文
