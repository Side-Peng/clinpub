# 选题挖掘工作流

## 目的

从临床数据表中挖掘研究选题，不执行统计分析或论文写作。三步流程：数据画像 -> 文献扫描 -> 课题生成。每步展示中间结果供用户反馈。

## 详细步骤

### 步骤 1：run_data_profile（优先：first）

读取用户提供的 CSV 或 XLSX，生成综合数据画像：

```bash
python scripts/data_profiler.py <filepath> --output idea/data_profile.json
```

画像包含：
1. **变量清单**：名称、数据类型、缺失率、唯一值数
2. **分布摘要**：连续变量五数概括，分类变量频率
3. **缺失模式**：高缺失（>20%）标记、中缺失（5-20%）提示
4. **相关矩阵**：数值变量 Spearman 相关（>30列时警告）
5. **变量角色检测**（自动推断）：

| 角色 | 名称模式 |
|------|----------|
| 结局变量 | outcome, 结局, diagnosis, status, death, event |
| 暴露/分组 | group, treatment, arm, exposure, 暴露, trt, 随机 |
| 时间变量 | time, follow, survival, months, days, 随访 |
| 协变量 | age, sex, gender, bmi, smoke 等 |
| 生物标志物 | biomarker, 蛋白, gene, serum, plasma, score 等 |

6. **研究类型预测**：

| 数据特征 | 建议设计 |
|----------|----------|
| 随机分组 + 基线 + 随访数据 | RCT |
| 事件时间 + 暴露分组 | 队列研究 |
| 病例/对照 + 匹配ID | 病例对照 |
| 单一时间点 + 暴露 + 结局 | 横断面 |
| 仅描述性变量，无分组 | 描述性 |
| 多个生物标志物 + 结局 | 标志物组合/诊断 |

7. **样本量评估**：

| 样本量 | 建议 |
|--------|------|
| <50 | 仅描述性 |
| 50-200 | 简单比较或描述性 |
| 200-500 | 可进行回归分析 |
| 500-2000 | 大多数分析方法适用 |
| >2000 | 复杂建模 + 亚组分析 |

向用户展示画像。确认变量角色后再继续。

> 自动检测的变量角色可能不准确——用户必须在继续前确认。

### 步骤 2：literature_scan_parallel（优先：high）

基于数据画像，分派并行 PubMed 检索——每个变量组一个子任务。

#### 前提——内置 PubMed 检索

`scripts/ncbi_search.py` 随管线内置，无需额外安装。

可选：设置 `NCBI_API_KEY` 环境变量提速（3req/s -> 10req/s）。

#### 阶段 1：变量分组

从 `idea/data_profile.json` 创建检索任务：

1. **疾病背景**：用户描述 + 变量名 -> 基础疾病关键词
2. **生物标志物/暴露变量**：每个检测到的标志物/暴露 -> 一个检索任务
3. **结局变量**：结局特定检索
4. **组合检索**：前 2-3 个变量对 -> 交叉参考检索

跳过：患病率 <5%、纯描述性（ID、日期）或已知协变量（age、sex）。

#### 阶段 2：并行分派

每个子任务的模板：

```
变量：{variable_name}
疾病背景：{disease_keywords}
检索词："{variable_name}" AND "{disease}" AND ("cohort" OR "RCT" OR "observational" OR "association")
附加过滤：近5年，英文，人类

任务：
1. 运行 PubMed 检索
2. 统计近5年论文数
3. 识别前3篇高影响力论文
4. 判断发表趋势
5. 识别研究空白

返回结构化摘要：
- 变量名
- 检索词
- 近5年论文数
- 代表性论文
- 发表趋势：上/下/稳
- 发现的空白
- 新颖度评分：高/中/已充分研究
```

#### 阶段 3：汇总结果

1. 汇总所有摘要到 `idea/literature_scan.md`
2. 交叉参考空白——发现**复合新颖性**
3. 按新颖度排序
4. 标记已充分研究变量——建议差异化策略

写入 `idea/literature_scan.md`：逐变量摘要表、跨变量空白分析、复合新颖性发现。

### 步骤 3：generate_topics（优先：high）

综合数据画像 + 文献扫描，生成 3-5 个候选课题。

选题策略：
- 大样本（>5000行）-> 优先队列或 RCT
- 多标志物（>10个）-> 优先标志物组合或 LASSO
- 无分组/结局变量 -> 仅描述性研究
- 用户指定方向 -> 优先匹配

写入完整报告到 `idea/选题报告.md`。末尾包含对比排序表。

### 步骤 4：generate_project_config（优先：high）

用户选定课题后，生成可直接启动管线的 `project_config.yml`。

触发条件：用户说"选第 N 个"或确认某课题。

写入 `idea/to_project_config.yml`（不覆盖已有配置）。

## 成功标准

- 数据画像生成（变量清单、缺失报告、研究类型预测、样本量评估）
- 文献扫描完成含空白分析（新颖度标注）
- 3-5 个候选课题含可行性评分、变量映射和目标期刊
- 课题对比排序表
- 用户已选定课题（或返回优化）
- `idea/to_project_config.yml` 生成
- 用户了解后续步骤
