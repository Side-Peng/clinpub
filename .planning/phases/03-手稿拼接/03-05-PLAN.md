**正文结构：**

### 第一章：引用总量约束（per D-17）
- 总引用量 30-55 篇范围制，此为硬约束（不可突破）
- 不设固定目标值（不做"must be 50"），讨论时根据实际情况确定
- 此范围适用于所有研究类型的 IMRAD 手稿

### 第二章：各段引用配比（per D-18）
- 引用配比作为弹性建议（非硬约束），各段可 ±20% 浮动
- 段配比表（与 reference-library.md §2.3 保持一致）：

| 段落 | 建议引用数 | 弹性范围（±20%） | 说明 |
|------|-----------|-----------------|------|
| Introduction | 10-15 | 8-18 | 背景(3-5) + 已知证据(3-5) + gap(2-3) + 目的(1) |
| Methods | 3-5 | 2-6 | 方法学来源、已发表协议、诊断标准引用 |
| Results | 0-3 | 0-4 | 仅当结果需要与已有文献对比时引用 |
| Discussion | 15-25 | 12-30 | 对比(8-12) + 机制(3-5) + 临床意义(2-3) + 局限(1-2) |

- 总量验证：各段实际引用数之和必须在 30-55 范围内。如果弹性调整导致超限，优先缩减 Discussion 段的引用量

### 第三章：文献年限策略（per D-21）
- 默认：近 5 年文献
- 例外：landmark 经典文献不受年限限制，即使超过 5 年也可引用
- landmark 判定标准：
  - 该领域首次报告该发现的原创论文
  - 被引 >1000 次的方法学论文
  - 临床指南引用的核心文献
  - 系统综述/Meta 分析中的关键文献
- 讨论时与用户确认是否有 landmark 例外需要添加

### 第四章：影响因子偏好（per D-21）
- IF 偏好无严格默认值，每次写作前与用户讨论确定
- 讨论参考：目标期刊的 IF 范围、领域内高 IF 期刊列表
- IF 偏好影响 reference-agent 搜索时的文献筛选，但不作为硬性排除条件（低 IF 但高相关性的文献仍可收录）

### 第五章：引用格式（per D-08）
- Vancouver 编号制，正文 [1][2]，末尾统一 References 区
- 具体格式规范见 `pipeline/references/reference-library.md` §2

### 第六章：讨论流程（per D-19, D-20）
- 引用策略讨论发生在 writing workflow 的 `discuss_citation_strategy` 步骤
- 讨论顺序：各段数量 → 年份范围 → IF 偏好
- 讨论结果写入 `project_config.yml` 的 `citation_strategy` 段
- 后续 `reference_pre_search` 和 `sequential_section_writing` 步骤读取此配置

### 第七章：与已有规范的层级关系
- `citation-strategy.md` 定义"量"的层面（多少篇、多新、多高 IF）
- `reference-library.md` 定义"质"的层面（JSON 格式、编号、去重）
- 两者互补，共同构成完整的引用管理体系
